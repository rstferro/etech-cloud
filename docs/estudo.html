TypeScript ⭐

TypeScript = JavaScript + tipos checados em tempo de escrita. A trilha completa está em docs/estudo/01-typescript.md — aqui vai o cheat-sheet.

Union types & narrowing

type DialogState = null | { product?: Product };
// "estreitar" antes de usar:
if (dialog) { dialog.product } // aqui o TS sabe que não é null

Record — objeto com chaves tipadas

type Accent = "primary" | "warning" | "success" | "cyan";
const accents: Record<Accent, string> = { primary: "..." }; // faltou chave? erro!

Utility types — o pulo do gato 🐈

UtilityFaz
Partial<T>tudo opcional
Pick / Omitincluir / excluir chaves
ReturnType<F>o tipo que a função retorna
Awaited<P>desembrulha uma Promise
// o exemplo mais avançado do SEU projeto (service-orders.ts):
type ServiceOrderWithCustomer =
  Awaited<ReturnType<typeof listServiceOrders>>[number];
// função → Promise → array → 1 item. O tipo se atualiza sozinho ✨

as const → union de um array

const SERVICE_STATUSES = ["RECEBIDO", "PRONTO"] as const;
type Status = (typeof SERVICE_STATUSES)[number]; // "RECEBIDO" | "PRONTO"

z.infer → Zod gera o tipo

const productSchema = z.object({ /* ... */ });
type ProductInput = z.infer<typeof productSchema>; // tipo de graça
💡 Dica: passe o mouse sobre qualquer tipo no VS Code pra ver o TS expandir o resultado. É a melhor forma de aprender.

Next.js App Router

A estrutura de pastas em src/app/ = as rotas.

Route groups (parênteses)

app/
├─ (auth)/login/page.tsx        → /login
└─ (dashboard)/
   ├─ layout.tsx                → casca protegida
   ├─ dashboard/page.tsx        → /dashboard
   └─ ordens/page.tsx           → /ordens

Parênteses organizam sem aparecer na URL. Assim o layout protegido envolve só a área logada.

Server busca, Client interage

// page.tsx (Server) — async, query direta, SEM "use client"
export default async function ProdutosPage() {
  const products = await listProducts();
  return <ProductsManager products={products} />; // passa pro client
}
🔑 Padrão de ouro: a página (server) busca os dados; o manager (client) cuida da interação. Você vê isso em produtos e ordens.

Server Actions & Zod

Mutations sem API REST + validação que vira tipo.

A receita de 4 passos (toda action segue)

"use server";
export async function createOrderAction(_prev, formData) {
  const session = await auth();                  // 1. logado?
  if (!session?.user) return { ok: false };
  const p = serviceOrderSchema.safeParse(...);     // 2. válido?
  if (!p.success) return { ok: false, error: ... };
  await createServiceOrder(p.data);              // 3. grava
  revalidate();                                   // 4. atualiza tela
  return { ok: true };
}

Zod — safeParse

const s = z.object({
  customerName: z.string().trim().min(2, "obrigatório"),
  price: z.coerce.number().min(0).optional(), // "150" → 150
});
⚠️ Tudo que vem de FormData é string. Por isso z.coerce.number() converte antes de validar.

Prisma 7 (ORM)

Schema → client type-safe → SQL por baixo. Banco: SQLite local (dev.db).

Comandos do dia a dia

npx prisma migrate dev --name init  # cria/aplica migration
npx prisma generate                 # gera o client TS
npx prisma db seed                  # roda prisma/seed.ts
npx prisma studio                   # UI pra ver o banco
⚠️ Pegadinha do Prisma 7: migrate dev NÃO gera o client nem roda o seed sozinho. Rode os dois à mão.

Autenticação (Auth.js v5)

Login por credenciais, sessão em JWT, senha com bcrypt.

Split config (edge vs servidor)

config.ts

Só tipos + callback authorized. Edge-safe (sem bcrypt/Prisma).

index.ts

NextAuth completo + provider Credentials (usa bcrypt + Prisma).

Validar a senha

authorize: async (cred) => {
  const user = await prisma.user.findUnique({ where:{ email } });
  if (!user) return null;
  const ok = await bcrypt.compare(password, user.passwordHash);
  return ok ? { id:user.id, role:user.role, ... } : null;
}
⚠️ Next 16 renomeou middleware → proxy (export default de uma função). A guarda authorized roda a cada request.

Tailwind CSS v4

Classes utilitárias no JSX. A v4 é "CSS-first": config dentro do globals.css.

@theme — tokens viram utilities

@import "tailwindcss";
@theme {
  --color-primary: #8c5aff;   /* → bg-primary, text-primary... */
  --color-surface: #0d0a14;
}

Lendo uma classe

bg-surface-2   → background var(--color-surface-2)
px-4 py-2.5    → padding
hover:bg-...   → no hover
sm: / lg:      → responsivo (mobile-first)
bg-warning/10  → cor com 10% de opacidade
🎨 Mudou o token --color-primary, mudou o app inteiro. É o que mantém o tema cyberpunk consistente.

React 19 — hooks

Componentes + estado. Hooks só funcionam em Client Components.

useActionState (novo no 19)

const [state, formAction, pending] = useActionState(createProductAction, undefined);
<form action={formAction}>
  {state?.error && <p>{state.error}</p>}
  <button disabled={pending}>{pending ? "Salvando..." : "Criar"}</button>
</form> // loading sem useState! ✨

useTransition (ações sem travar)

startTransition(() => { moveOrderAction(id, col, orderedIds); });
// persiste o drag-and-drop sem congelar a UI

useEffect (sincronizar)

useEffect(() => { if (state?.ok) onDone(); }, [state, onDone]);
// o array no fim = dependências (quando re-rodar)
